[[overview-http-verbs]]
== HTTP verbs

|===
| Verb | Usage

| `GET`
| Used to retrieve a resource

| `POST`
| Used to create a new resource

| `PATCH`
| Used to update an existing resource, including partial updates

| `DELETE`
| Used to delete an existing resource
|===

[[overview-http-status-codes]]
== HTTP status codes

|===
| Status code | Usage

| `200 OK`
| The request completed successfully

| `201 Created`
| A new resource has been created successfully. The resource's URI is available from the response's
`Location` header

| `204 No Content`
| An update to an existing resource has been applied successfully

| `400 Bad Request`
| The request was malformed. The response body will // include an error providing further information

| `404 Not Found`
| The requested resource did not exist
|===

[[resources-settings]]
== Settings

=== Save new Settings

A 'POST' request is used to save the settings.
When channels are committed, the old channels should be overwritten.
When only one channel should change, use the Channel Endpoints.

==== Request structure

include::{snippets}/save-settings/request-fields.adoc[]

==== Example request

include::{snippets}/save-settings/http-request.adoc[]

==== JSON Request Schema

[source,JSON]
----
{
  "$schema": "http://json-schema.org/draft-04/schema#",
  "id": "/",
  "type": "object",
  "properties": {
    "wlan": {
      "id": "wlan",
      "type": "object",
      "properties": {
        "ssid": {
          "id": "ssid",
          "type": "string"
        },
        "pass": {
          "id": "pass",
          "type": "string"
        },
        "wlanSecurity": {
          "id": "wlanSecurity",
          "type": "string"
        }
      },
      "required": [
        "ssid",
        "pass"
      ]
    },
    "controllerAlias": {
      "id": "controllerAlias",
      "type": "string"
    },
    "clientId": {
      "id": "clientId",
      "type": "string"
    }
  },
  "required": [
    "controllerAlias",
    "clientId"
  ]
}
----
